## 1. The right architecture

### A.1. Canonical source of truth
Use **GitHub** as the real working project space.

Why:
- ChatGPT can connect to GitHub and search selected repositories. ([help.openai.com](https://help.openai.com/en/articles/11145903-connecting-github-to-chatgpt))
- GitHub gives you version control, diffs, history, branches, and an easy shared file structure.
- Both ChatGPT and Claude can work well from repo files.

### A.2. What Notion should be
Use **Notion as a mirror and mobile dashboard**, not as the canonical editable source.

Why:
- Notion’s built-in GitHub synced databases are **one-way only**.
- They are for **GitHub PRs and issues**, not arbitrary markdown docs or your custom apartment tracker files. ([notion.com](https://www.notion.com/help/synced-databases))

### A.3. How to sync GitHub to Notion
Use:
- a structured tracker file in the repo, ideally `CSV` or `JSON`
- a small Python sync script
- a GitHub Actions workflow that pushes updates into a Notion database via the Notion API

This is supported by:
- Notion API support for creating pages and database rows, and creating databases under a parent page. ([developers.notion.com](https://developers.notion.com/reference/post-page))
- GitHub Actions workflows that run YAML-defined automation from the repo. ([docs.github.com](https://docs.github.com/actions))
